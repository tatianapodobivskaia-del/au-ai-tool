Let's Get Set Up

The first 15 minutes of the workshop. Your instructor is demoing on the projector — follow along on your laptop. Don't panic if you fall behind on one step; a neighbor or the instructor will help. By the end of this block everyone in the room is ready to build.

⏱ 15 minutes 🧑‍🏫 Follow the instructor 💻 Mac or Windows

You do not need to know any code. Everyone starts from zero today — business and IT students alike. The only terminal moment in the whole workshop happens in Step 2 of this block, and it's one command.

If you get stuck at any point: raise your hand, check the Stuck? box under each step, or ask the neighbor who looks like they got it. Help each other. That's how this works.

1

Install Visual Studio Code

~4 min

VS Code is Microsoft's free code editor — the tool we'll use for the rest of the workshop. You don't need to write any code today, just know how to open it and click a few panels.

What to do

  1. Open a browser. Go to code.visualstudio.com.
  2. Click the big blue Download button. The site auto-detects your operating system.
  3. Open the installer file you just downloaded.
  4. Accept all the defaults (Next → Next → Install → Finish). They're sensible.
🍎 On macOS

The download is a .zip. Double-click it, then drag Visual Studio Code into your Applications folder.

🪟 On Windows

During install, make sure the checkbox for "Add to PATH" is checked. It's on by default — just don't uncheck it.

✓
Open VS Code. You should see a welcome screen that says "Visual Studio Code" across the top.
Stuck?

"The application can't be opened" (Mac) → right-click the icon → Open → confirm.

Installer asks for admin password → that's normal, enter it.

2

Install Git

~3 min

Git is the version-control tool everything else runs on. VS Code uses it behind the scenes. Follow your platform's section only — the process is different on Mac and Windows.

🍎 On macOS

Nothing to download. Just ask the system for Git:

  1. Open Terminal (⌘+Space → type "Terminal").
  2. Type git --version → press Enter.
  3. A dialog: "Install Command Line Developer Tools?" → click Install.
  4. Wait ~2 minutes for it to finish.
🪟 On Windows

Download the installer:

  1. Go to git-scm.com/download/win.
  2. Download starts automatically. Run the .exe.
  3. Accept all defaults (Next → Next → Install).
  4. Click Finish when done.
✓
Open a fresh terminal (Mac: Terminal · Windows: Command Prompt or PowerShell) and type git --version. You should see something like git version 2.45.0. The exact number doesn't matter.
Heads up This is the only moment we touch a terminal. You won't need it again unless you want to.
Stuck?

Windows says "git is not recognized" → close and reopen the terminal. If still broken, restart the computer.

Mac says "xcrun: error" → re-run in Terminal: xcode-select --install.

3

Create a GitHub Account

~4 min

GitHub is where your project will live online. If you already have a GitHub account, skip to Step 4.

What to do

  1. Go to github.com/signup.
  2. Enter your email. Use a personal email (Gmail, Outlook, iCloud) — not a school or work email you might lose access to. This account is yours for life.
  3. Create a password.
  4. Pick a username. This matters — it'll be part of your public URL (github.com/YOUR-USERNAME) and your live app's address. Pick something you'd put on a résumé: your real name or a clean variation (jane-smith, jsmith-ai). Avoid joke names.
  5. Solve the puzzle / captcha.
  6. GitHub sends a verification code to your email. Open your email, copy the code, paste it in.
  7. Answer the onboarding questions however you want. Pick the Free plan.
✓
You're on your GitHub dashboard at github.com, and your username is visible in the top-right corner.
Important Verify your email right now. An unverified account can't create repositories — which would break the rest of the workshop for you. The email arrives in 10–30 seconds; check spam if needed.
Stuck?

Username taken → add your field (-ai, -dev) or initials.

Didn't get the verification email → check spam. Then go to github.com/settings/emails → Resend verification email.

4

Sign In to GitHub from VS Code

~2 min

This is the most important step. Once VS Code is signed in to your GitHub account, everything else just works — no passwords, no tokens. Skip this step and you'll fight authentication popups all workshop.

What to do

  1. Open VS Code.
  2. Look at the very bottom-left corner. Find the small person icon 👤.
  3. Click it → menu appears → click "Sign in with GitHub to use GitHub…"
  4. Your browser opens a GitHub authorization page.
  5. Sign in if asked, then click the green Authorize Visual-Studio-Code button.
  6. Browser says "You can close this tab now." → return to VS Code.
What success looks like # Bottom-left corner of VS Code:
👤 your-github-username ← this is the signal you're signed in
✓
Your GitHub username is visible next to the person icon at the bottom-left of VS Code. Authentication is now handled forever.
Stuck?

No person icon visible → open the Command Palette (⇧⌘P / Ctrl+Shift+P) and search Sign in with GitHub.

Browser never opened → it may have opened behind other windows. Check your taskbar/dock.

5

Install the GitHub Extension

~2 min

This extension turns VS Code into a full collaboration workbench — pull requests, reviews, and issues, all without leaving the editor.

What to do

  1. In VS Code's left sidebar, click the Extensions icon (four squares, 4th icon from the top).
  2. In the search box, type: GitHub Pull Requests
  3. The first result should be "GitHub Pull Requests and Issues" — publisher: GitHub (blue verified checkmark ✓).
  4. Click the blue Install button.
  5. Wait ~10 seconds.
Pick the right one Several extensions have similar names. Publisher must be GitHub with the blue verified checkmark. Don't install third-party alternatives.
What success looks like # A new icon appears in the left sidebar:
🐙 GitHub ← the octocat — we'll use this in Step 11 to open a PR
✓
The GitHub octocat icon appears in your VS Code sidebar (usually between the Source Control and Extensions icons).
Stuck?

Icon didn't appear → reload VS Code: Command Palette (⇧⌘P / Ctrl+Shift+P) → Developer: Reload Window.

Didn't finish?

That's okay. This block is 15 minutes but some installs are slower than others. The workshop won't leave you behind. Tell your instructor which step you're stuck on — they'll help you catch up during the next break, or pair you with a neighbor.

If your laptop really can't install something (admin-locked, no disk space, etc.), you can do the entire workshop in a browser at vscode.dev. It's not ideal but it works.

All five boxes checked?

You're ready for the rest of the workshop. When everyone in the room is set up, the instructor will close the laptops for 15 minutes and walk us through the Concept Primer — what Git is, what GitHub is, and the difference between them.
